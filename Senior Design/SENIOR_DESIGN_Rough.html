Abstract

Multiple online databases such as Brenda, METACYC, Kegg, contain collected kinetic parameters of enzymes for many studied model organisms. While many data points exist, they lack coverage at a network and species level, creating nodes with no validated data, which decreases model prediction capability. The objective is to obtain, curate and clean the kinetic parameters (target) data and the metabolite and protein structural (features) data for enzymes in human central metabolism. Using kinetic data for central metabolism in humans scraped from Brenda and collecting its protein and metabolite features, the model out of Linear Regression, Elastic Net Regression and Random Forest ML algorithms that performed the best at predicting KM parameters was the Random Forest with an accuracy of 0.01469. The inability to create a robust model suggests that the data and/or the algorithms chosen are insufficient to characterize the more complex mechanisms relating structure and function, suggesting future work should attempt to obtain larger, more complete sets of data and/or apply other machine learning algorithms.

Background

Systems biology uses a reiterative process consisting of obtaining data, analyzing the data, making inductions/predictions to improve computational models and gain mechanistic understanding of the biological characteristics of metabolic networks. The chemical workhorses in the body are enzymes which have been optimized over time by evolution. However, which enzyme features are selected for by evolution to create a more efficient enzyme is not completely understood. In order for scientists to be able to create efficient industrial bioprocesses, comprehensive models that have predictive power and provide a deeper mechanistic understanding of biological networks are needed to maximize yields of bioproducts. With respect to progress in enzymatic dynamical systems over the last few decades, the traditional Michaelis-Menten approximation is still the best known approach to the enzyme kinetics; however, with such complex model structure, kinetics and regulation of cell metabolism can hardly be understood. Currently, there are limited workflows for obtaining and processing large amounts of proteomic data as it is computationally expensive. Since separate databases use non-standardized ids and irregular data structures, kinetic data is also difficult to obtain on a network scale because no single location exists to capture, consolidate, and standardize the format of the kinetic data.

Objectives

The objective is to obtain, curate and clean kinetic target data and metabolite and protein features for enzymes in human central metabolism to apply Elastic Net Regression and Random Forest ML algorithms to obtain important features for highly efficient enzymes. There exists a need for a workflow/pipeline to help gap-fill and predict kinetic parameters for different enzymes. Because structural properties are available for most enzymes even if the kinetic data is not, the goal of this project is to use structural properties of enzymes to predict the kinetic parameters of the reaction.

Project Overview

The Goal is to:

  • Clean kinetic parameter (target) data
  • Curate metabolite and protein structural (features) data
  • Find structural properties that correlate with high efficacy for enzymes
  • Gapfill and predict kinetic parameters for different enzymes using Random Forest and P-Elastic Net Regression Algorithms.
  • Obtain more complete genome-scale metabolic (GEM) models.

Methods

  • Since there are more than 50,000 reactions with multiple data points for each reaction/enzyme that require curation work, coding scripts were written to help to match the kinetic data to enzyme reaction and structural properties.
  • Enzyme commission (EC) number, which are used to identify reactions, can be shared by a broad class of enzymes, as such the kinetic data had to be manually curated to match to a reaction from the scraped data to RECON3D.
  • Chemaxonâ€™s Marvin, a chemically desktop toolkit, was used to calculate the feature properties of different metabolites/proteins
  • The structural properties of the protein and metabolites were ordered into two separate matrices. The metabolite data was aggregated into values per reaction such that each kinetic data point had its corresponding features.
  • The data will be preprocessed using Principal Component Analysis (PCA) to reduce the dimensionality of the matrix and identify important features.
  • The matrix with aggregated kinetic and structural data will then be fed into two separate machine learning algorithms, Random Forest and Elastic Net Regression to find correlation between highly efficient enzymes and protein and metabolite features.
